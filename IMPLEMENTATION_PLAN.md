# План реализации приложения для тестирования ПДД Грузии

## Анализ структуры данных

### Поля вопроса (ru.json):
- **id** - уникальный идентификатор вопроса
- **question** - текст вопроса
- **answer_1, answer_2, answer_3, answer_4** - варианты ответов (до 4)
- **correct_answer** - номер правильного ответа ("1", "2", "3" или "4")
- **hasImg** - флаг наличия изображения (0 или 1)
- **img** - путь к изображению
- **question_explained** - объяснение к вопросу
- **category** - ID категории

### Категории (categories.json):
- Объект с ключами-ID и значениями-названиями категорий

## План реализации

### Этап 1: Базовая инфраструктура
1. **Настройка SCSS**
   - Создание глобальных стилей и переменных
   - Настройка миксинов и базовых стилей

2. **Типы TypeScript**
   - Интерфейсы для Question, Category
   - Типы для LocalStorage данных

3. **Сервисы**
   - LocalStorageService с методами для работы с данными
   - QuestionService для загрузки и фильтрации вопросов
   - CategoryService для работы с категориями

### Этап 2: Роутинг
1. **Настройка React Router**
   - Основные маршруты
   - Вложенные маршруты для категорий

### Этап 3: Компоненты Layout
1. **Header** - верхняя панель управления
2. **MainLayout** - двухколоночная структура

### Этап 4: Компоненты категорий
1. **CategoryList** - список всех категорий
2. **CategoryItem** - элемент категории с:
   - Индикатором изученности
   - Счетчиками и ссылками на специальные списки

### Этап 5: Компоненты вопросов
1. **QuestionList** - список вопросов с пагинацией
2. **QuestionCard** - карточка вопроса с:
   - Номером вопроса
   - Изображением (или заглушкой)
   - Текстом вопроса
   - Подсказкой в тултипе
3. **AnswerOption** - вариант ответа с:
   - Нумерацией 1-4
   - Подсветкой правильного/неправильного ответа

### Этап 6: UI компоненты
1. **Pagination** - компонент пагинации
2. **Tooltip** - всплывающая подсказка

### Этап 7: Страницы
1. **HomePage** - главная страница
2. **CategoryPage** - страница категории
3. **FavoritesPage** - избранные вопросы
4. **KnownPage** - вопросы "точно знаю"
5. **HardPage** - сложные вопросы

### Этап 8: Функциональность
1. Загрузка и отображение вопросов
2. Фильтрация по категориям
3. Система меток (избранное, знаю, сложное)
4. Сохранение состояния в LocalStorage
5. Пагинация с настраиваемым количеством элементов

## Технические детали

### LocalStorage структура:
- `settings`: настройки приложения
- `complete`: массив ID изученных категорий
- `[categoryId]-favorite`: избранные вопросы категории
- `[categoryId]-known`: известные вопросы категории
- `[categoryId]-hard`: сложные вопросы категории

### Используемые библиотеки:
- **React Router** - маршрутизация
- **lucide-react** - иконки
- **SCSS** - стилизация

## Дизайн-система

### Цветовая схема:
- Основной фон: #2C2C2C (темный)
- Карточки: #FFFFFF
- Правильный ответ: #4CAF50 (зеленый)
- Неправильный ответ: #FF5252 (красный)
- Серый (не изучено): #9E9E9E
- Избранное: #FFC107 (желтый)

### Типографика:
- Основной шрифт: -apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto
- Размеры: 14px (основной), 12px (мелкий), 16px (заголовки)
